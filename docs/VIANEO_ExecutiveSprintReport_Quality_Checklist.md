# Vianeo Executive Sprint Report - Quality Checklist

**Version:** 1.0
**Purpose:** Ensure Executive Sprint Reports meet quality standards before delivery
**Last Updated:** December 8, 2025

---

## Pre-Generation Checklist

### Data Completeness

- [ ] Project metadata complete (name, PI, institution, dates, author)
- [ ] All scores present and valid (0.0-5.0 range)
- [ ] Overall Vianeo score calculated correctly (weighted average)
- [ ] Market maturity score present
- [ ] Status determination correct (GO/CONDITIONAL PROCEED/NO-GO)
- [ ] All 5 dimensions have complete data (score, status, summary, strengths, gaps)
- [ ] Minimum strengths per dimension met (3-7 items)
- [ ] Minimum gaps per dimension met (3-5 items)
- [ ] All personas have profiles, needs, and validation requirements (3-4 personas)
- [ ] All ecosystem relationships have type, criticality, and status (5-8 relationships)
- [ ] All priorities have owners, timelines, and items (3 immediate, 3 short-term, 3 medium-term)
- [ ] All risks have impact ratings and mitigation strategies (5-7 risks)
- [ ] Conclusion has 4 paragraphs
- [ ] Next review section complete

### Data Quality

- [ ] No placeholder text ([Insert X], [TBD], etc.) remains
- [ ] Dates properly formatted (e.g., "December 8, 2025")
- [ ] Scores formatted consistently (e.g., "4.1 / 5.0" or "4.1/5.0")
- [ ] Status values exact case ("PASS" not "Pass", "FAIL" not "Fail")
- [ ] Weights include % symbol ("15%" not "15" or "0.15")
- [ ] Interview counts specific (e.g., "15-20 interviews" not "interviews needed")
- [ ] Evidence includes metrics where possible (not just "strong" or "good")
- [ ] Gaps explain implications (not just "X missing" but "X missing, which means Y")

## Content Quality Checklist

### Executive Summary

- [ ] Project overview has exactly 3 paragraphs
- [ ] Project overview word count: 250-350 words
- [ ] All 5 dimensions present in key findings table
- [ ] Dimension scores match between dashboard and detailed sections
- [ ] Primary recommendation has clear status statement
- [ ] 4-6 validation gaps listed
- [ ] 4-6 immediate next steps listed
- [ ] All validation gaps are specific (not "needs validation" but "zero customer interviews conducted")

### Business Model Overview

- [ ] Value proposition is 2-3 sentences, 50-75 words
- [ ] Core differentiation has 4-6 specific items
- [ ] Target segments table has 6-10 rows
- [ ] Each segment has specific characteristics (not generic)
- [ ] Revenue model type specified (hybrid/subscription/transactional)
- [ ] Revenue components include percentage splits
- [ ] Pricing warning present if pricing unvalidated

### Evaluation Results

- [ ] All 5 dimensions assessed
- [ ] Each dimension has score, status, threshold
- [ ] Each dimension has 1-2 sentence summary
- [ ] Legitimacy has 5-6 strengths, 3-4 gaps
- [ ] Desirability has 5-6 strengths, 3-4 gaps
- [ ] Acceptability has 5-6 strengths, 3-4 gaps
- [ ] Feasibility has 5-6 strengths, 3-4 gaps
- [ ] Viability has 5-6 strengths, 3-4 gaps
- [ ] All strengths include specific evidence (metrics, names, numbers)
- [ ] All gaps include implications (not just stating the gap)
- [ ] Tone is balanced (both strengths and gaps present, not one-sided)
- [ ] No generic language ("good team" → "13+ doctorate-level researchers with...")

### Stakeholder Analysis

- [ ] 3-4 personas present
- [ ] Each persona has specific role name (not "User" or "Customer")
- [ ] Each persona has 1-sentence profile
- [ ] Each persona has 3-4 specific needs
- [ ] Each persona has validation requirements with interview count
- [ ] Ecosystem relationships table has 5-8 rows
- [ ] Each relationship has criticality rating (Critical/High/Medium/Low)
- [ ] Each relationship has status (Active/Informal/Not Engaged/etc.)

### Recommendations

- [ ] 3 immediate priorities (0-30 days) present
- [ ] 3 short-term priorities (30-90 days) present
- [ ] 3 medium-term priorities (90-180 days) present
- [ ] Each immediate/short-term priority has owner
- [ ] Each immediate/short-term priority has timeline
- [ ] Each immediate/short-term priority has 4-7 items
- [ ] Medium-term priorities have 1-2 paragraph descriptions (condensed format)
- [ ] Risk mitigation table has 5-7 rows
- [ ] Each risk is specific scenario (not "things might go wrong")
- [ ] Each risk has impact rating (Critical/High/Medium/Low)
- [ ] Each mitigation strategy is actionable (not "monitor situation")

### Conclusion

- [ ] Exactly 4 paragraphs present
- [ ] Paragraph 1 focuses on strengths and achievements
- [ ] Paragraph 2 focuses on gaps and challenges
- [ ] Paragraph 3 begins with "The path forward is clear:" (bold)
- [ ] Paragraph 4 begins with "The assessment recommends [RECOMMENDATION]" (recommendation bold)
- [ ] Next review section has timing
- [ ] Next review has 4-6 expected deliverables
- [ ] Next review has 4-6 success criteria with thresholds

## Formatting Checklist (DOCX)

### Document Structure

- [ ] Cover page present with title, tagline, subtitle, metadata table
- [ ] All 6 sections present in correct order
- [ ] 11 page breaks at correct locations (after cover, before each major section, before each dimension 3.2-3.5, before 5.3, before Section 6)
- [ ] Headers appear on all pages except cover
- [ ] Footers with page numbers appear on all pages
- [ ] Page numbers are sequential and correct

### Typography

- [ ] Cover title is 28pt bold, primary blue (#1E3A5F), centered
- [ ] Cover subtitle is 22pt bold, primary blue, centered
- [ ] Tagline is 14pt italic, gray, centered
- [ ] Assessment type is 12pt, secondary blue, centered
- [ ] Heading 1 is 16pt bold, primary blue
- [ ] Heading 2 is 14pt bold, black
- [ ] Heading 3 is 12pt bold, black
- [ ] Body text is 11pt regular, black
- [ ] Metadata is 9pt regular, light gray
- [ ] Font family is Arial throughout

### Tables

- [ ] All tables have header rows with blue background (#1E3A5F)
- [ ] All table headers have white text, bold, 11pt, centered
- [ ] All tables have alternating row shading (white and light blue #E8F4F8)
- [ ] Column widths specified for all tables
- [ ] Column widths sum to 6.5 inches (or close to full text width)
- [ ] Score dashboard: 3 equal columns
- [ ] Key findings: 4 columns (1.8", 0.8", 1.0", 2.9")
- [ ] Target segments: 2 columns (2.0", 4.5")
- [ ] Ecosystem relationships: 4 columns (1.8", 1.5", 1.5", 1.7")
- [ ] Risk mitigation: 3 columns (2.0", 1.0", 3.5")

### Color Coding

- [ ] PASS status is green (#28A745)
- [ ] FAIL status is red (#DC3545)
- [ ] CONDITIONAL status is yellow (#FFC107) where applicable
- [ ] Critical impact/criticality is red (#DC3545)
- [ ] High impact/criticality is yellow (#FFC107)
- [ ] Medium impact/criticality is green (#28A745)
- [ ] Low impact/criticality is gray (#6C757D)
- [ ] Active/Funded status is green (#28A745)
- [ ] Informal/Not Engaged status is yellow (#FFC107)
- [ ] All colors match specification exactly (hex codes, not "blue" or "green")

### Lists

- [ ] All bullet lists use Word's built-in bullet formatting (not unicode • or -)
- [ ] All numbered lists use Word's built-in numbering
- [ ] List indentation is consistent (0.25" from margin, 0.25" hanging)
- [ ] No manual numbering (1., 2., 3. as text)

## Content Validation Checklist

### Evidence-Based Writing

- [ ] All claims supported by evidence (not "good team" → "13+ doctorate-level researchers")
- [ ] Metrics included where possible (scores, counts, percentages, dollar amounts)
- [ ] Sources implicitly cited (no footnotes, but authority clear from context)
- [ ] Tone is objective (not "we believe" or "it seems")
- [ ] Language is professional (not academic jargon or marketing fluff)

### Honesty & Balance

- [ ] Strengths are genuine (not aspirational)
- [ ] Gaps are honest (not euphemistic)
- [ ] Every dimension has both strengths and gaps (balanced)
- [ ] Recommendations are directive (not "consider" but "execute")
- [ ] Unvalidated assumptions clearly marked as "(unvalidated)" or "(hypothesized)"
- [ ] Interview counts accurate (if zero, state "zero interviews conducted" not "validation needed")

### Specificity

- [ ] No vague language ("strong", "good", "significant", "several")
- [ ] Specific numbers provided (not "many researchers" → "13+ researchers")
- [ ] Specific roles named (not "stakeholders" → "Nuclear Medicine Physicians")
- [ ] Specific timelines (not "soon" → "within 60 days")
- [ ] Specific metrics (not "improved outcomes" → "20-29% failure rate reduction")

## Scoring Verification

### Overall Vianeo Score Calculation

Verify weighted average is correct:

```
Overall = (Legitimacy × 0.15) + (Desirability × 0.25) + (Acceptability × 0.20) + (Feasibility × 0.20) + (Viability × 0.20)
```

- [ ] Calculation verified manually
- [ ] Score matches data file
- [ ] Score rounded to one decimal place (X.X)

### Threshold Compliance

- [ ] Legitimacy meets 3.0 threshold (if PASS)
- [ ] Desirability meets 3.5 threshold (if PASS) - **HIGHER threshold**
- [ ] Acceptability meets 3.0 threshold (if PASS)
- [ ] Feasibility meets 3.0 threshold (if PASS)
- [ ] Viability meets 3.0 threshold (if PASS)

### Status Determination

- [ ] Status is GO if overall ≥ 4.0 and market maturity ≥ 3.2
- [ ] Status is CONDITIONAL PROCEED if overall 3.0-3.9 OR market maturity < 3.2
- [ ] Status is NO-GO if overall < 3.0 OR any dimension fails threshold
- [ ] Status rationale explained in primary recommendation

## Common Errors to Check

### Frequent Content Errors

- [ ] No "TBD" or placeholder text remains
- [ ] No copy-paste errors from previous projects (wrong project name, etc.)
- [ ] No orphaned references to non-existent sections
- [ ] No broken internal links (in Markdown)
- [ ] No inconsistent terminology (same concept called different things)

### Frequent Formatting Errors

- [ ] No em dashes (—) anywhere (use commas or periods)
- [ ] No unicode bullets (•) in DOCX (use Word formatting)
- [ ] No manual page breaks (use section breaks where appropriate)
- [ ] No inconsistent spacing (6pt after paragraphs should be consistent)
- [ ] No font mixing (Arial throughout, no Calibri or Times New Roman)

### Frequent Data Errors

- [ ] No score mismatches (dashboard vs. detailed sections)
- [ ] No dimension weights summing to ≠ 100%
- [ ] No PASS status with score below threshold
- [ ] No interview counts that don't match narrative (e.g., "zero interviews" but persona marked "validated")
- [ ] No dates in the future (report date should be today or in past)
