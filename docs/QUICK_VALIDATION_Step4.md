## 4. MEANS ANALYSIS (6 items)

**Human Means:**
- [ ] Team capabilities listed with specificity
- [ ] Each means marked as Differentiating: Yes/No

**Physical/Intellectual Means:**
- [ ] Technology, IP, assets identified
- [ ] Each means marked as Differentiating: Yes/No

- [ ] Current funding status documented
- [ ] Each means marked as Differentiating: Yes/No

## 5. DIFFERENTIATION (4 items)

- [ ] At least 1-2 differentiating means identified
- [ ] Differentiation is genuine (not table stakes)

- [ ] Problem validated externally (not just team belief)
- [ ] Validation sources cited
- [ ] Confidence levels assigned (L1/L2/L3)
- [ ] Gaps clearly documented

## INSTANT RED FLAGS

**Stop immediately if you see:**
- ‚ùå Solution embedded in problem: "Need our AI platform to..."
- ‚ùå No differentiating means identified
- ‚ùå All validation is internal ("team believes")
- ‚ùå Problem is vague: "inefficient processes"
- ‚ùå Missing quantification of problem impact
- ‚ùå Field of application = solution description

---

## COMMON FIXES (2-minute corrections)

**Solution in problem statement?**
- ‚ùå "Healthcare providers need our scheduling software"
- ‚úÖ "Healthcare providers spend 12+ hours/week on manual scheduling"
- Test: Remove your product ‚Üí still a valid problem?

**No differentiating means?**
- Review team backgrounds for unique expertise
- Check IP/technology for novel elements
- Look for exclusive partnerships or access

**Validation only internal?**
- Add "[L1 - requires external validation]" label
- Identify specific interviews or research needed
- Don't claim validation that doesn't exist

**Problem too vague?**
- Add specific numbers (time, cost, frequency)
- Name specific affected population
- Cite source of problem data
