## COMMON FIXES (2-minute corrections)

**Need >60 characters?**

Shortening techniques:
- Remove filler: "very", "really", "that", "which"
- Use "&" instead of "and"
- Use "/" instead of "or"
- Remove articles: "the", "a", "an"
- Abbreviate: "%" not "percent", "#" not "number"
- Active voice: "Reduce time" not "Time needs to be reduced"

**Example:**
- ‚ùå (78 chars) "Students need to be able to track their learning progress over time easily"
- ‚úÖ (57 chars) "Track learning progress over time w/ minimal effort"

**Overlapping needs?**
- Test: Could both needs be met by same feature?
- If YES ‚Üí Merge or make more distinct
- Each need should address different aspect

**Solution in need statement?**
- ‚ùå "Need mobile app for scheduling"
- ‚úÖ "Schedule appointments quickly from anywhere"
- Test: Remove technology/product ‚Üí still a need?

**Missing "Doing Nothing"?**
- Always include as Existing Solution #1
- Description: "Status quo - no dedicated tool, manual process"
- Limitations: Why current approach fails

## THE 60-CHARACTER RULE

**Why it matters:**
- Forces clarity and specificity
- Prevents run-on vague statements
- Enables visual matrix display
- Professional presentation standard

**Character counting:**
- Includes spaces and punctuation
- Use character counter tool
- 60 chars ‚âà 8-10 words ‚âà 1 short sentence

## NEEDS CATEGORIZATION

**Tasks (Jobs to be done):**
- What users are trying to accomplish
- Productive outcomes they seek
- Example: "Create weekly lesson plans in under 2 hours"

**Pains (Problems to eliminate):**
- Frustrations, obstacles, inefficiencies
- What slows them down or creates errors
- Example: "Eliminate duplicate data entry across 3 systems"

**Expectations (Desired outcomes):**
- Quality standards, reliability requirements
- How they want to feel during/after
- Example: "Receive instant feedback on quiz performance"

**Distribution:** Aim for roughly 4 tasks, 4 pains, 2 expectations
