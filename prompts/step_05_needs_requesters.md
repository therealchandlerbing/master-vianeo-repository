# Step 5: Needs & Requesters

## Overview
This step applies the WHO/WHAT/WHY/HOW framework to assess desirability and validate user needs.

## Purpose
Systematically identify and validate the needs of target users and stakeholders to ensure the solution addresses real, valuable problems.

## Framework Components

### WHO - Target Users
- User identification
- Segmentation
- Prioritization
- Personas

### WHAT - Needs Definition
- Explicit needs
- Latent needs
- Job to be done
- Pain points

### WHY - Underlying Motivations
- Core drivers
- Value expectations
- Decision criteria
- Success metrics

### HOW - Current Solutions
- Existing alternatives
- Workarounds
- Competitive solutions
- Gaps in current offerings

## Desirability Assessment
- Need intensity
- Market size
- Willingness to pay
- Switching costs

## Output
- Validated needs list
- User priorities
- Desirability score
- Design implications
